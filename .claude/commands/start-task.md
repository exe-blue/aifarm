# /start-task - 새 작업 시작

새로운 작업을 시작할 때 이 커맨드를 실행하여 체계적으로 진행합니다.

## 작업 시작 체크리스트

### 1. DOAI.md 읽기
```bash
cat DOAI.md | head -100
```
- 최근 실수 기록 확인
- DO NOT / MUST DO 규칙 숙지

### 2. 작업 정의
다음 질문에 답하세요:

| 항목 | 답변 |
|------|------|
| 이 작업의 목표는? | |
| 성공 기준은? | |
| 검증 방법은? | |
| 예상 소요 시간은? | (2-4시간 이내여야 함) |
| 이 작업만 할 것인가? | (발산 금지) |

### 3. 의존성 확인
```bash
# 현재 브랜치
git branch --show-current

# 최신 상태 동기화
git pull origin main

# 관련 서비스 실행 상태
curl -sf http://localhost:8006/health && echo "✅ Persona Service OK"
```

### 4. 작업 범위 확정
- [ ] 수정할 파일 목록 정의
- [ ] 각 파일의 변경 범위 정의
- [ ] 테스트 방법 정의

## 작업 진행 원칙

### DO
- 구현 → 검증 → 수정 사이클 반복
- 작은 단위로 커밋
- 막히면 DOAI.md 확인

### DON'T
- 여러 기능 동시 작업
- 검증 없이 다음 단계
- 범위 밖 작업 (발산)

## 완료 시
```
/verify  # 검증 실행
/commit-pr  # 커밋 및 PR
```

## 작업 템플릿

```markdown
## 작업: [제목]

### 목표
[구체적인 목표 기술]

### 성공 기준
- [ ] 기준 1
- [ ] 기준 2

### 검증 방법
1. [검증 단계 1]
2. [검증 단계 2]

### 수정 파일
- `path/to/file1.py` - [변경 내용]
- `path/to/file2.js` - [변경 내용]

### 진행 상황
- [x] 완료된 항목
- [ ] 진행 중
- [ ] 대기 중
```
