# /handoff - 작업 인수인계

세션 종료 또는 다른 에이전트에게 작업을 넘길 때 이 커맨드를 실행합니다.

## 핸드오프 체크리스트

### 1. 현재 상태 정리
- [ ] 모든 변경 사항 커밋됨
- [ ] 진행 중인 작업 없음 (또는 명확히 기록됨)
- [ ] DOAI.md 업데이트됨

### 2. 핸드오프 문서 작성

```markdown
## 핸드오프 리포트

### 날짜/시간
[YYYY-MM-DD HH:MM]

### 완료된 작업
1. [완료된 작업 1]
2. [완료된 작업 2]

### 미완료 작업
1. [미완료 작업 1] - 진행률: X%, 남은 작업: [설명]
2. [미완료 작업 2] - 진행률: X%, 남은 작업: [설명]

### 발생한 이슈
- [이슈 1]: [상태 및 해결 방법]
- [이슈 2]: [상태 및 해결 방법]

### 다음 작업자에게
- [주의사항 1]
- [주의사항 2]

### 관련 파일
- `path/to/modified/file.py` - [변경 내용 요약]

### 환경 상태
- 서비스 실행 중: [목록]
- DB 상태: [정상/이슈]
- 테스트 결과: [통과/실패]
```

## 핸드오프 저장 위치

### 짧은 메모
`deploy/HANDOFF_PROMPT.md` 업데이트

### 상세 기록
`docs/handoffs/YYYY-MM-DD_[작업명].md` 생성

## 자동 체크

```bash
# 커밋되지 않은 변경 확인
git status

# 최근 커밋 확인
git log --oneline -5

# 서비스 상태 확인
curl -sf http://localhost:8006/health && echo "✅ Persona Service"
curl -sf http://localhost:8000/health && echo "✅ API Gateway"
```

## 핸드오프 예시

```markdown
## 핸드오프 리포트

### 날짜/시간
2025-01-06 15:30

### 완료된 작업
1. DOAI.md 파일 생성 - 프로젝트 AI 가이드
2. .claude/commands/ 슬래시 커맨드 설정
3. 검증 스크립트 통합

### 미완료 작업
없음

### 발생한 이슈
- 없음

### 다음 작업자에게
- MVP 범위에 집중할 것 (6개 활동 중 1개만 먼저)
- 검증 스크립트 실행 후 작업 진행 권장

### 환경 상태
- 서비스 실행 중: 없음 (수동 실행 필요)
- 테스트: 미실행
```
